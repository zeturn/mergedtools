## ✨ 升级亮点

### 从 7 种 → 27 种编码格式！

**新增 20 种编码格式:**

#### 🧱 标准 Base 编码 (15种新增)
- **Base2 (Binary)** - 二进制编码，机器码表示
- **Base4 (DNA)** - DNA 编码，ACGT 字符集
- **Base8 (Octal)** - 八进制，Unix 文件权限
- **Base10 (Decimal)** - 十进制，人类常用
- **Base12** - 十二进制，时间系统
- **Base16 (Hex)** - 十六进制，哈希值/颜色代码
- **Base20** - 二十进制，玛雅进制
- **Base24** - 24进制，紧凑编码
- **Base26** - 字母表，列号表示
- **Base36** - 数字+字母，URL短链
- **Base40** - ASCII 子集
- **Base52** - 纯字母编码
- **Base56** - 去混淆字符
- **Base60** - 六十进制，时间制度
- **Base70** - 增强可读性

#### 🔥 高密度编码 (3种新增)
- **Base91** - 高效编码，约82%密度
- **Base94** - 全 ASCII 字符 (0x21-0x7E)
- **Base66** - 扩展 Base64

#### 💎 特殊变体 (2种新增)
- **Base32 Hex** - Base32 十六进制变体
- **Base64 URL** - URL 安全的 Base64

## 📚 使用文档

### 🔥 最热门的 5 种编码

1. **Base64** - Web 标准，使用最广泛
2. **Base32** - Google 2FA、OTP 密钥
3. **Base58** - 比特币地址、IPFS 哈希
4. **Base62** - URL 短链、唯一 ID
5. **Base45** - 健康码、二维码

### 💡 选择建议

**需要URL安全？**
→ Base64URL, Base62, Base58

**需要人工输入？**
→ Base32 (不区分大小写)

**需要最高密度？**
→ Base91, Base94, Base85

**需要标准规范？**
→ Base64 (RFC 4648), Base32 (RFC 4648), Base45 (RFC 9285)

**需要去除混淆字符？**
→ Base58 (去除 0/O/I/l)

## ⚠️ 注意事项

1. **大文件处理**
   - 建议文件大小 < 10MB
   - 大文件可能导致浏览器卡顿

2. **编码选择**
   - Base2/Base4 会产生非常长的输出
   - 实验性编码兼容性可能有限

3. **数据安全**
   - 所有处理在浏览器本地完成
   - 数据不会上传到服务器

4. **浏览器兼容性**
   - 现代浏览器均支持
   - 某些老旧浏览器可能不支持部分API
